# 2장. 자바스크립트란?

## 2.1 자바스크립트의 탄생

- 탄생 배경
    - 웹페이지의 보조적인 기능을 수행하기 위해 도입(1995)
- 이름 변천사
    - 모카(1996, 3월) → 라이브스크립트(1996, 9월) → 자바스크립트(1996, 12월)

## 2.2 자바스크립트의 표준화

- **크로스 브라우징 이슈**
    - 브라우저에 따라 웹페이지가 정상적으로 동작하지 않는 이슈 발생
    - 모든 브라우저에서 정상적으로 동작하는 웹페이지를 개발하는데에 큰 어려움
- **ECMAScript**
    - 모든 브라우저에서 정상적으로 동작하는 표준화된 자바스크립트의 필요성이 대두됨
    - ECMA 인터내셔널에 자바스크립트 표준화 요청 → 1997년 1월, 자바스크립트 초판(ECMAScript 1) 사양 완성
    - 2015년에 공개된 ECMAScript 6은 `let/const 키워드, 화살표 함수, 클래스, 모듈` 등 범용 프로그래밍 언어로서 갖춰야 할 기능들이 대거 도입됨

## 2.3 자바스크립트 성장의 역사

- 초창기의 자바스크립트는 웹페이지의 보조적인 기능을 수행하기 위해 한정적인 용도로 사용됨
- 웹 서버에서 실행 / 브라우저는 서버로부터 전달받은 HTML과 CSS를 단순히 렌더링하는 수준

### 2.3.1 Ajax(XMLHttpRequest)

- 1999년
- 자바스크립트를 이용해 서버와 브라우저가 **비동기 방식**으로 데이터 교환 가능
- 서버로부터 필요한 데이터만 전송받아 변경해야 하는 부분만 한정적으로 렌더링하는 방식
    - 웹 브라우저에서도 데스크톱 애플리케이션과 유사한 빠른 성능과 부드러운 화면 전환 가능
    - 2005년, **구글 맵스**

### 2.3.2 jQuery

- 2006년 등장
- DOM을 더욱 쉽게 제어, 크로스 브라우징 이슈 어느정도 해결
- 자바스크립트보다 배우기 쉽고 직관적

### 2.3.3 V8 자바스크립트 엔진

- 2008년
- 더욱 빠르게 동작하는 자바스크립트 엔진의 필요성에 부합하는 빠른 성능 보유
- UX 제공
- V8의 등장으로 웹 서버 → 클라이언트로 이동

### 2.3.4 Node.js

- 2009년, 구글 V8 자바스크립트 엔진으로 빌드된 자바스크립트 런타임 환경
- 브라우저 이외의 환경에서도 동작할 수 있도록 자바스크립트 엔진을 브라우저에서 독립시킨 자바스크립트 실행 환경
    - 서버 사이드 애플리케이션 개발에 주로 사용됨
    - 모듈, 파일 시스템, HTTP 등 빌트인 API 제공
    - 프론트엔드, 백엔드 영역 모두 자바스크립트를 사용할 수 있음 (동형성)
- **비동기 I/O 지원, 단일 스레드 이벤트 루프 기반**
    - 요청 처리 성능 ദ്ദിᐢ- ̫-ᐢ₎
    - I/O가 빈번하게 발생하는 SPA(Single Page Application)에 적합
    - CPU 사용률이 높은 애플리케이션엔 권장하지 않음

### 2.3.5 SPA 프레임워크

- 개발 규모, 복잡도 상승 & 아키텍처 구축 어려움 → 필연적으로 프레임워크 등장
- CBD(Component based development) 방법론을 기반으로 하는 SPA
    - Angular, React, Vue.js, Svelte 등…

## 2.4 자바스크립트와 ECMAScript

- 자바스크립트의 표준 사양인 ECMA-262를 의미
    - 핵심 문법 규정(프로그래밍 언어의 값, 타입, 객체와 프로퍼티, 함수, 표준 빌트인 객체 등…)
    - 각 브라우저 제조사는 ECMAScript 사양을 준수해서 브라우저에 내장되는 자바스크립트 엔진 구현
- 자바스크립트는 ECMAScript를 아우르는 개념
    - ECMAScript + 브라우저가 별도 지원하는 **클라이언트 사이드 Web API**
        - DOM, BOM, Canvas, XMLHttpRequest, fetch, … 등

## 2.5 자바스크립트의 특징

- **웹 브라우저에서 동작하는 유일한 프로그래밍 언어**
- 기본 문법은 C, 자바와 유사 / 셀프에서는 프로토타입 기반 상속 / 스킴(Scheme)에서는 일급함수 개념 차용
- **인터프리터 언어**(별도 컴파일 작업 수행X)
    - 대부분의 자바스크립트 엔진은 인터프리터 + 컴파일러 장점을 결합한 형태로 속도 느린 인터프리터 단점 해결
- 명령형, 함수형, 프로토타입 기반 객체지향 프로그래밍을 지원하는 **멀티 패러다임 프로그래밍 언어**
- **프로토타입 기반의 객체지향 언어**가 맞다!!
